Overview

Postgres Database based tool for processing griptester readings.

- fits grip tester readings to network to make table fitted

-uses benchmark runs and sections to correct for season

-gets readings for 10m pieces of network

-outputs hmdif files.

How to Use

under 'setup':

-connect to a database

-setup the database if this has not already been done.

-the network is in table -network. Import this into qgis then set network layer to this.

-readings are in table -r. Import this into qgis then set readings layer to this.

under 'upload':

-upload the readings

-ensure information in run_info table is correct

under 'fitting':

Create route table for each run. The 'autofit run' button will do most of this. Check rows highlighted yellow. Rows highlighted red overlap and are wrong. Use the map to check for bad gps.

under 'output':

download hmd files

under 'coverage':

use 'search hmds' button to find missing requested sections. Check table r on the map to make sure the section wasn't surveyed .Then look for a reason on the map or in the summary sheet.
